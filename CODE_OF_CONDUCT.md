# Code of Conduct

## 我们希望的协作方式

`皮匠` 是一个还在快速收敛主线的开源项目。

这意味着我们尤其重视：

- 诚实反馈，而不是包装进展
- 可验证结论，而不是空泛争论
- 对新人友好，但对技术判断保持严格
- 聚焦问题本身，不攻击人

## 期望行为

- 用事实、日志、复现步骤和最小样例讨论问题
- 尊重不同背景的贡献者
- 在 review、issue 和讨论中保持清晰、克制和可执行
- 承认不确定性，并愿意修正自己的判断

## 不可接受的行为

- 人身攻击、羞辱、骚扰
- 恶意刷屏、挑衅、引战
- 明知含有 secrets 仍公开贴出
- 故意误导项目状态、验证结果或支持边界

## 维护者责任

维护者有责任：

- 保护讨论环境
- 及时纠正明显越界行为
- 在需要时收敛讨论范围，避免主线失焦
- 对支持边界、路线图和拒收标准保持一致口径

## 执行

若发生明显越界行为，维护者可以：

- 提醒
- 隐藏或删除内容
- 暂停互动
- 拒绝继续协作

## 适用范围

本准则适用于：

- GitHub issues
- pull requests
- discussions
- 与仓库相关的贡献协作空间
