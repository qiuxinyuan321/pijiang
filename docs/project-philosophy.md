# 皮匠的核心理念与演化方式

`皮匠` 想解决的，不只是“让模型更强回答一次”，而是让用户拥有一套**超越单次大模型回答的系统级能力**。

如果用一句话概括：

> **我们希望通过这套系统，让你拥有“超越大模型的大模型”。**

这里的“超越”不是说单个模型被神化升级，而是说用户获得了一套比“单模型单次回答”更强的决策系统。

## 黑盒问题

现在多数 AI 产出，对用户来说仍然像黑盒。

- 你通常只看到了一个答案
- 却不知道它是怎么来的
- 也不知道还有哪些路线被错过了
- 更不知道这个答案在不同强模型或不同入口下，是否还有明显优化空间

这会直接削弱用户的判断力。尤其在“打地基”“定方向”“选架构”“做长期取舍”这类问题上，只有结果、没有过程，往往就意味着你只能接受一次黑盒输出。

## 摇奖模型

为了帮助理解，可以把单次模型回答理解成一次“摇奖”。

- 单次回答像一次抽样
- 强模型的优势很多时候只是“奖池更优”
- 但奖池更优，不等于用户自动拥有了“取舍权”

这里的“摇奖”是用户体验层的比喻，不是说模型能力等于随机。它想强调的是：

> **单次回答天然带有采样性、视角性和盲区。**

如果用户只能接受一次输出，那么无论模型多强，过程仍然是黑盒的。

## 起源实验

`皮匠` 的灵感不是先有一个抽象理论，再去找场景套上去。它来自一个很具体的真实观察：

- 在相同提示词下，`Codex + GPT-5.4` 和 `Claude Code + GPT-5.4` 产出了两版不同方案
- 两版方案都不差，但关注点、结构和取舍明显不同
- 双方互相借鉴之后，仍然还有 `7` 条可以继续优化的地方

这个观察特别关键，因为它说明问题不只是“谁更强”。

更核心的是：

- 单次输出本身就有采样性
- 不同入口和不同工作流会放大不同盲区
- 即使强模型之间互相借鉴，仍然存在显著的优化空间

所以在“打地基”“定方向”这类问题上，用户不能只接受一次黑盒抽样。你真正需要的是：

- 多版方案
- 互相借鉴
- 彼此攻击
- 最后再做融合和取舍

## 皮匠的制度解法

`皮匠` 的答案不是“多摇几次奖”这么简单，而是把这个过程制度化。

它的制度解法包括：

- 多模型
  - 不同模型位承担不同分析职责
- 多职责
  - controller、planning、search、marshal、chaos、skeptic、fusion 各自承担明确任务
- 对抗
  - 不是只求共识，而是允许质疑、红队、破局
- 借鉴
  - 不同方案之间可以互相吸收优点
- 融合
  - 最终不是堆答案，而是进入决策账本和终版草案
- truth audit
  - 不让“看起来成功”掩盖真实问题

换句话说：

> **皮匠让用户不再只能接受一次黑盒摇奖，而是拥有在多次方案对照中选择更优结果的能力。**

这就是“超越大模型的大模型”的关键。

## 为什么这是用户级深度思考

`皮匠` 不是企业咨询替代品，也不是只给少数人玩的高配实验品。

它想做的是把原本只有少数人靠大量时间、高昂成本、多轮讨论才能获得的能力，压到用户侧：

- 多轮比较
- 多路对照
- 互相借鉴
- 互相反驳
- 最后做取舍

这就是我们说的“用户级深度思考”。

它的目标不是让用户更依赖黑盒，而是让用户重新拥有“取舍权”。

## 为什么它会继续进化

AI 时代最强的能力之一，不只是更强回答，而是会复盘自己、会升级自己、会用升级后的自己继续讨论自己。

`皮匠` 的进化靠的不是单模型自言自语，而是一套制度化机制：

- 真实会议
- truth audit
- regression cases
- benchmark
- delta-report
- next-iteration brief

这意味着：

- 系统不是一次性设计出来的
- 失败不是噪音，而是下一轮进化的燃料
- 升级后的议会还会继续讨论议会自身

这也是为什么我们把“自举”“自省”看得很重。对这个项目来说，真正的进化不是换一句提示词，而是让多模型集思广益、互相纠偏之后，把能力一轮轮压进系统。

> **议会制度不是装饰，它本身就是这个项目的进化引擎。**
